The Challenge

One queue, every kind of inquiry.

Atlas HXM is a global Employer of Record platform — it legally employs workers in 160+ countries on behalf of client companies. Its "Contact Us" form is a front door for everything: real buyers asking to speak with sales, job seekers, employment-verification requests, press, and partnership pitches, all landing in the same queue.

That mix created two expensive problems. First, sales-ready leads waited in line behind noise — a buyer asking for "EOR services in South Africa" got the same treatment as a résumé drop. Second, and more damaging for the automation program: nobody could answer the question "what has this actually made us?" Marketing automation without revenue attribution is a cost center in every budget conversation.

The project had two halves: build the automation, then prove its economic value with numbers a CFO would accept.

The approach: classify once, act everywhere

Instead of routing raw form text to humans, an AI agent reads each submission and makes one decision — what is this person actually asking for? — and every downstream action (CRM update, lifecycle stage, partner routing, auto-reply) flows from that single classification.

For measurement, the standard shortcut would be a HubSpot attribution report. That was rejected in favor of execution-level attribution: pull every individual run from the n8n API, extract the exact contact that came through, then match those contacts against CRM deals with a strict rule — a deal counts only if it was created on or after the day that contact came through the workflow. This deliberately excludes pre-existing deals on returning contacts — including a $337K closed-won deal that predated the workflow by one day — making the final number conservative and defensible.

What Was Built

An 18-node workflow, plus a proof layer.

The automation runs in real time on every form submission. The measurement pipeline runs on top of it, turning execution logs into auditable revenue numbers.

01 · Intake

Real-time webhook

A webhook node receives every HubSpot form submission the moment it lands.

02 · Classification

AI intent agent

An Azure OpenAI agent classifies each message into one of seven intents — Speak with Sales, Career, Employment Verification, Partnership, General, Press, Support.

03 · Business Logic

Deterministic rules

A code node decides what actually happens: does the lifecycle stage move to MQL? Does this route to partnerships? The AI classifies; the code decides.

04 · System of Record

HubSpot writes

HubSpot nodes write the classification, update the lifecycle stage, and log notes — the CRM is always the source of truth.

05 · Response

RAG-grounded replies

A second AI agent, grounded by an Azure AI Search vector store over Atlas knowledge, drafts a contextual reply to sales-ready leads — referencing the country and service the prospect asked about.

06 · Routing

Partner path

Partnership inquiries get a HubSpot note and a routed email via Microsoft Outlook — off the sales queue entirely.

The measurement pipeline

  • Resumable ingestion. A cursor-checkpointed Python client pulled the workflow's complete 166-run execution history from the n8n API — any interruption resumed cleanly instead of restarting.
  • Execution parsing. A parser walked each run's node-level output and extracted the contact, the AI's classification, whether the MQL node fired, and any errors.
  • CRM matching. The HubSpot MCP matched 121 unique emails against live CRM records (116 found), then pulled every deal with amounts, stages, close dates, and WSE counts.
  • Attribution + audit. Attribution logic separated the 29 workflow-attributed deals from 5 pre-existing ones, and a formula-driven Excel workbook made every number traceable down to the execution ID.

Systems Thinking

Decisions that made the number defensible.

  • Attribution integrity over headline size. The "deal created on/after submission" rule cost the headline number over $1.1M in pre-existing deals — including a $337K win that closed one day before the workflow's first run. The smaller number survives due diligence; an inflated one doesn't.
  • Failure-aware ingestion. The n8n pull was built as a resumable, cursor-checkpointed client; every batch persisted state, so interruptions resumed instead of restarting.
  • Test-data hygiene. 22 of 166 runs were internal test submissions. They're flagged and visible in the model rather than silently deleted — transparency beats tidy data.
  • Honest gaps, priced anyway. 9 attributed deals had no amount in HubSpot yet. Rather than ignore them or guess, the model derives a median value-per-WSE ($62,453) from the 18 deals that do have amounts, applies it to the 43 estimated WSEs, and exposes the rate as a single editable assumption cell.
  • Everything cites a source. Every contact row carries its n8n execution ID; every deal its HubSpot record ID. Any number in the summary traces to a raw API record.
