Inbound Automation Workflow

Overview

A global HR-tech (EOR) platform's website contact form is a firehose: genuine buyers, partnership pitches, job seekers, vendors, press, and support requests all arrive through one endpoint. This live n8n workflow classifies every submission with an enterprise LLM into one of seven canonical intents (Speak with Sales, Partnership, Career, Customer Support, Press, Employment Verification, General), stamps the right CRM lifecycle stage, routes partnership leads to the alliances owner, and — for genuine sales intent — drafts and logs a human-sounding follow-up email. A hard override honors the reason the visitor actually picked on the form, so the AI never second-guesses an explicit choice.

The Problem

Inbound form submissions mixed high-intent buyers with vendor spam, job applications, and support tickets, all landing in one place for a human to read, categorize, and route by hand. Real "Speak with Sales" leads waited behind noise, partnership inquiries got misrouted, and lifecycle stages were stamped inconsistently — so reporting on true inbound demand was unreliable.

How It Works

  1. Webhook intake — a CRM form submission POSTs into n8n (a workflow-automation platform) via a header-authenticated webhook the instant someone submits.
  2. AI intent classification — an enterprise LLM reads the message, the form's selected reason, and the source/page context, and returns exactly one of seven canonical categories. A hard-override rule fires first: if the visitor explicitly selected a valid reason on the form, that choice wins and the AI doesn't override it.
  3. Reason-for-contact write-back — the classified intent is written to the contact's reason-for-contact property via the CRM API.
  4. Deterministic lifecycle mapping — a hand-written JavaScript node maps intent → CRM lifecycle stage (Sales → MQL, Partnership → partnership stage, everything else → an "other" stage), normalizes the original form selection, and emits routing flags — with logic to detect partnership signals even in re-classified submissions.
  5. Partnership routing — genuine partnership inquiries are stamped to the alliances owner and can trigger a personalized, on-brand reply drafted by a second AI agent (writing as the named partnerships lead), sent via email.
  6. Sales follow-up drafting — for "Speak with Sales" intent, an AI agent writes only the body of a warm, human follow-up email (calm, non-pitchy, acknowledges their message, asks for country only if missing, points to a calendar link) and stamps it to a response property for the rep.
  7. Shared error handling — the workflow is its own error-workflow target, so failures surface consistently rather than silently.

Setup Process

  1. Create/confirm the CRM properties the workflow writes (reason-for-contact, lifecycle stages, response fields).
  2. Import the workflow JSON into n8n; bind CRM, LLM, and email credentials.
  3. Point the CRM form's webhook (header-auth) at the n8n trigger URL.
  4. Confirm the seven canonical category strings and the hard-override list match your CRM's picklist values exactly.
  5. Dry-run with sample payloads for each intent (sales / partnership / vendor spam / job applicant); verify lifecycle stamping and that vendor pitches are filtered to "General" rather than "Sales."
  6. Activate; wire the workflow into your shared error-handler.

What You Need

  • n8n cloud workspace with webhooks enabled
  • Enterprise LLM deployment + credential
  • CRM private-app token (contacts read-write, lifecycle, owners)
  • Email OAuth connection
  • A defined intent taxonomy + owner assignments · Time-to-deploy: ~2–3 hours

Results

  • Live in production on the company's contact form.
  • Seven-way AI intent classification with an explicit hard-override so a visitor's stated reason is never overridden by the model.
  • Deterministic intent→lifecycle mapping keeps CRM reporting on true inbound demand honest.
  • Vendor/recruiter pitches routed to "General" (not "Sales"), keeping the MQL pipeline clean.

Challenges & Fixes

  • AI over-classifying vendor pitches as sales leads → explicit vendor/outreach trigger-phrase rules force "General"; only genuine service demand becomes "Speak with Sales."
  • Model second-guessing a visitor's explicit form selection → a hard-override rule evaluated first returns the selected reason verbatim, no analysis.
  • Partnership inquiries arriving via re-classified paths → the JS mapper detects partnership signals in message, form name, and original label — not just the AI output.
  • Lifecycle regressions from blind writes → lifecycle only updates when the target differs from current, so a re-submission never downgrades a contact.
