The LLM maps messy CRM text to canonical sectors; a deterministic JavaScript engine issues every verdict. Systems that survive failure

A multi-agent post-call automation was failing 21 of 25 runs — a dead Azure AI Search index left four AI agents with no context. I didn't recreate the fragile index. I designed the failure point out: extract countries → ISO-3 codes → deterministic blob fetch. No search, no credentials to rotate, graceful fallback. Context dropped from 882KB to 295KB while groundedness improved. 0/25 failing → stable end-to-end.

The ROSTR Framework

I didn't just build agent systems — I formalized the architecture. The ROSTR Framework is a published, citable paper covering four components (PAL, RAG DAL, NPAO, Rostr Hub) and a 5D phase taxonomy for production-grade multi-agent orchestration. It's the engine behind Gency AI and a contribution to the field.

PAL — Prompt Abstraction Layer

Transforms loose natural language into strict agent runtime manifests through a 5-stage compilation pipeline. No raw prompts — every invocation flows through compiled manifests.

RAG DAL — Dynamic Acquisition Layer

Multi-pass retrieval across three credibility tiers (academic → editorial → community). Confidence scoring weights source count, consistency, tier distribution, and recency.

NPAO — Phase-Aware Orchestration

5D phase taxonomy classifies every task (PreD → Debugging). 4D priority scoring. Agent allocation via vector similarity + specialization overlap + load scoring.

Rostr Hub — Persistent State

Four-level state management (session → project → org → agent). Knowledge compounds across sessions instead of evaporating.
